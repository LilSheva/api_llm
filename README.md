# Утилита анализа логов через локальную LLM (CLI Log Analyzer)

Консольный Python-скрипт для эффективного построчного чтения больших файлов логов, их упаковки в псевдо-XML структуру и отправки чанками в локальную LLM (OpenAI-совместимое API) для выявления ошибок, аномалий и предупреждений.

Скрипт написан **исключительно на стандартных библиотеках Python** (без внешних зависимостей типа `requests`), что позволяет легко компилировать его в переносимый исполняемый `.exe` файл для Windows без необходимости устанавливать Python на целевых машинах.

---

## 🚀 Основные возможности

1. **Эффективность по памяти**: Чтение гигабайтных файлов логов через Python-генератор. Файл не загружается целиком в ОЗУ, считывание происходит строго порциями (чанками) по $N$ строк (по умолчанию 400).
2. **Внешний файл настроек (`config.json`)**: Все параметры (URL, модель, лимиты, системные промпты, приветствие) хранятся во внешнем JSON-файле. Можно менять инструкции для LLM и параметры сети без перекомпиляции EXE.
3. **Автогенерация конфигурации**: Если файла `config.json` нет рядом со скриптом/EXE, программа автоматически создаст его со значениями по умолчанию.
4. **Интерактивное приветствие**: При запуске утилиты без параметров (например, двойным щелчком или без флагов) выводится наглядное приветственное сообщение с объяснением XML-структуры тегов и примером запроса.
5. **Чистый вывод**: Отладочные логи работы пишутся в стандартный поток ошибок (`stderr`), а итоговый JSON-результат — в стандартный вывод (`stdout`). Это позволяет использовать перенаправление потоков или утилиты типа `jq` (например: `log_analyzer.exe -f app.log > results.json`).
6. **Два варианта разметки (XML)**:
   - **Вариант 1 (API-Native)**: Инструкции передаются в системной роли (`system`), логи — в пользовательской роли (`user`).
   - **Вариант 2 (Explicit XML inside User role)**: Все данные объединяются в одну пользовательскую роль (`user`) с явными тегами `<SystemPrompt>` и `<UserPrompt>`.

## 🛠 Настройки (`config.json`)

Файл конфигурации автоматически генерируется при первом запуске утилиты в папке со скриптом или исполняемым `.exe` файлом.

Пример структуры `config.json`:
```json
{
  "api_url": "http://localhost:8080/v1/chat/completions",
  "model": "local-model",
  "limit": 400,
  "variant": "1",
  "timeout": 120,
  "temperature": 0.0,
  "system_instructions": "Ты системный аналитик...",
  "welcome_message": "..."
}
```

*Вы можете свободно изменять текст `system_instructions` в файле конфигурации для тонкой настройки качества анализа и логики ответов модели.*

## 💻 Аргументы командной строки (CLI)

Аргументы командной строки имеют более высокий приоритет и перегружают значения, указанные в `config.json`.

| Аргумент | Альтернатива | Описание |
| :--- | :--- | :--- |
| `-f` | `--file` | **[Обязательный]** Путь к анализируемому файлу логов. |
| `-u` | `--api-url` | URL API модели (по умолчанию из `config.json`). |
| `-l` | `--limit` | Размер одного чанка в строках (по умолчанию из `config.json`). |
| `-v` | `--variant` | Вариант упаковки XML: `"1"` или `"2"` (по умолчанию из `config.json`). |
| `-m` | `--model` | Имя модели для передачи в payload (по умолчанию из `config.json`). |
| `-t` | `--timeout` | Таймаут сетевого запроса в секундах (по умолчанию из `config.json`). |
| `--dry-run` | *нет* | Тестовый режим: выводит в консоль готовый JSON-запрос для первого чанка логов и завершает работу (без отправки в API). Очень удобно для проверки разметки. |
| `-d` | `--verbose` | Включает подробное логирование уровня DEBUG в `stderr`. |

## 📦 Сборка в один файл `.exe` для Windows

Чтобы пользователи могли запускать утилиту без установки Python, скомпилируйте её в один независимый файл `.exe`.

### Шаг 1. Установите PyInstaller
Запустите в терминале:
```bash
pip install pyinstaller
```

### Шаг 2. Выполните сборку
Перейдите в папку со скриптом и выполните команду:
```bash
pyinstaller --onefile --name log_analyzer analyze_logs.py
```

### Шаг 3. Заберите готовый файл
После завершения процесса скомпилированный файл `log_analyzer.exe` появится в созданной папке `dist/`. 
Скопируйте `log_analyzer.exe` в любое удобное место.

*При первом запуске `log_analyzer.exe` автоматически сгенерирует рядом файл настроек `config.json`.*

## 📝 Примеры использования

### 1. Интерактивный запуск (вывод справки и приветствия)
Просто запустите скрипт или EXE без параметров:
```bash
python analyze_logs.py
# или
log_analyzer.exe
```

### 2. Обычный анализ логов с записью результата в JSON файл
Служебные сообщения будут выводиться в консоль (`stderr`), а итоговый JSON сохранится в `results.json`:
```bash
python analyze_logs.py -f app.log > results.json
# или
log_analyzer.exe -f app.log > results.json
```

### 3. Запуск с перегрузкой параметров
Анализировать чанками по 200 строк с использованием Варианта упаковки 2:
```bash
python analyze_logs.py -f app.log -l 200 -v 2 > output.json
```

### 4. Отладка шаблона запроса (Dry-Run)
Вывести структуру запроса (XML-инкапсуляцию) в консоль для проверки:
```bash
python analyze_logs.py -f app.log --dry-run
```
