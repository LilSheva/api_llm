# Log Analyzer Utility

Скрипт для построчного чтения больших файлов логов, упаковки их в XML-структуру и пакетного анализа через API локальной LLM.

Написан на стандартной библиотеке Python без внешних зависимостей. Поддерживает простую компиляцию в исполняемый `.exe` файл для Windows с помощью `PyInstaller`.

## Возможности

* **Память-эффективное чтение**: построчная обработка файлов через генераторы. Лог не загружается в ОЗУ целиком, а отправляется чанками по $N$ строк (по умолчанию 400).
* **Внешняя конфигурация**: все сетевые параметры, настройки модели и системные инструкции вынесены в `config.json`. При отсутствии файла он будет автоматически создан с базовыми параметрами.
* **Разделение потоков**: отладочные логи записываются в `stderr`, а итоговый JSON-результат — в `stdout`, что позволяет перенаправлять вывод в файлы или использовать конвейеры (`jq` и др.).

## Конфигурация (`config.json`)

```json
{
  "api_url": "http://localhost:8080/v1/chat/completions",
  "model": "Meta-Llama-3.1-8B-Instruct-Q6_K.gguf",
  "limit": 400,
  "timeout": 120,
  "temperature": 0.0,
  "system_instructions": "..."
}
```

## CLI Параметры

| Флаг | Описание |
| :--- | :--- |
| `-f`, `--file` | **[Обязательный]** Путь к файлу логов для анализа. |
| `-u`, `--api-url` | URL-адрес эндпоинта LLM API. |
| `-l`, `--limit` | Количество строк лога в одном чанке. |
| `-m`, `--model` | Название целевой модели в запросе. |
| `-t`, `--timeout` | Время ожидания ответа API в секундах. |

## Сборка в один файл `.exe`

1. Установите PyInstaller:
   ```bash
   pip install pyinstaller
   ```
2. Выполните сборку:
   ```bash
   pyinstaller --onefile --name log_analyzer analyze.py
   ```
   Исполняемый файл появится в папке `dist/log_analyzer.exe`.

## Примеры использования

```bash
# Стандартный запуск с выводом на экран
python analyze.py -f app.log

# Сохранение результата анализа в файл JSON
python analyze.py -f app.log > report.json
```
